Get a list of people who selected a given class as their nth choice (rank)

Throws InvalidClassRankException if the given rank is not in 0 to 5, inclusive.

Will crash if course code does not exist.

The first call to this function after a course/people load might take longer. All subsequent calls use cached results and will return instantaneously.

The frontend should NOT default to calling this function and use the length of the iterable as the number of choices. Query this only when the user want to see this info. This function is slower than getNumChoicesForClassRank.

Implementation

Set<String> getPeopleForClassRank(String course, int rank) {
  if (rank < 0 || rank > 5) {
    throw InvalidClassRankException(rank: rank);
  }
  if (rank == 0) {
    return _data[course]!.firstChoices;
  } else {
    return _data[course]!.backups[rank - 1];
  }
}
